##Objects and Classes
----------------------
- Distinction between objects (instances of classes) and
classes (blueprints for creating objects)


##class
-------
Katgoul blli:
class hiya blueprint (plan) li katsawr lina structure dyal object, chno khaso ykoun fih, chno properties (variables/attributes) dyalo, w chno methods (fonctions) li y9dar ydir.
Structure dyal object: kifach data kaytkhzn f object.
Properties/priorities: hiya attributes li katmss object.
Methods: functions li object y9dar ydirhom.

##object
--------
Object huwa ŸÜÿ≥ÿÆÿ© 7aqiqiya (instance) mn class, 3ndo attributes dyalo
w y9dar yst3ml methods dyal class.

Class = plan
Object = dar 7aqiqi mabni mn had plan

An object is a model of something that has data and
behaviors associated with it, akin to physical objects we can
sense and manipulate.
- Object-oriented means being directed toward modeling
these objects, providing a method for understanding and
constructing complex systems.

##Inheritance
---------------
Inheritance hiya ila class jdida (child) katwarri l-attributes w methods 
dyal class okhra (parent), w t9dar tzed chi haja jdida.

: A mechanism where a new class can inherit attributes and
behaviors from an existing class

##Abstraction
----------------
- Abstraction is used to simplify complex systems by
focusing on relevant details for a given task while ignoring
unnecessary information.
